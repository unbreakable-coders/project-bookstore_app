# Git Workflow команди

Цей документ описує, як ми працюємо з Git та GitHub у проєкті.

---

## Основні гілки

У нас є **дві long-lived гілки**:

- `main` — стабільна, релізна гілка
  - деплой йде саме з неї
  - тільки перевірений, робочий код
- `develop` — робоча гілка
  - сюди зливаються всі виконані задачі через Pull Request
  - саме з неї кожен розробник відгалужується для своїх задач

> **Важливо:** прямі пуші в `main` і `develop` заборонені.  
> Усі зміни потрапляють туди **лише через PR**.

---
